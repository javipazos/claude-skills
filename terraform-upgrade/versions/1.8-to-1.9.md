# Terraform 1.8 → 1.9

Invalid import blocks now error instead of being silently ignored.

## Breaking Changes

### Invalid Import Blocks Now Error
- Import blocks pointing to nonexistent modules were silently ignored in prior versions
- In 1.9, these produce errors — they must be fixed or removed

### Test Provider Version Constraints
- Version constraints can no longer be specified within provider blocks in `.tftest.hcl` files
- Constraints must be in the main configuration's `required_providers`

## Automated Fixes

### Find invalid import blocks
```
Grep pattern: import\s*\{  in *.tf files
```
For each import block, verify the `to` target refers to a resource that exists in the configuration. Remove or fix any that reference nonexistent modules or resources.

### Move test provider constraints
```
Grep pattern: version\s*=  in *.tftest.hcl files
```
If found inside provider blocks in test files, move the version constraint to the main configuration's `required_providers` block and remove it from the test file.

## Manual Steps

### Review import blocks
The developer should verify that all import blocks still reference valid resources. Some may be leftover from previous refactoring.

## Verification

```bash
terraform validate
terraform plan
```
