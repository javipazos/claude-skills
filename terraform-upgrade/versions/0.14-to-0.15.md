# Terraform 0.14 → 0.15

Major cleanup release. Many features deprecated since 0.12 are now removed.

## Recommended Tool

```bash
terraform init
terraform fmt
```

No dedicated upgrade command. Apply automated fixes before running init.

## Breaking Changes

### Removed Functions
- `list()` removed → use `tolist([...])`
- `map()` removed → use `tomap({...})`
- These were deprecated since 0.12

### Removed Provisioners
- Built-in vendor provisioners removed: `chef`, `habitat`, `puppet`, `salt-masterless`
- No automatic replacement — requires architectural decision

### Removed Backends
- `atlas` backend removed (deprecated since 0.12)
- `gcs` backend: `path` argument removed → use `prefix`

### Removed CLI Options
- `terraform init`: `-lock`, `-lock-timeout`, `-verify-plugins=false`, `-get-plugins=false` removed
- `terraform destroy`: `-force` removed → use `-auto-approve`
- `terraform validate`: `-var` and `-var-file` removed (had no effect since 0.12)
- Trailing `[DIR]` argument removed from all commands → use `-chdir` global option

### Configuration Syntax
- `ignore_changes = ["*"]` removed → use `ignore_changes = all`
- Quoted variable type constraints removed: `"map"` → `map(string)`, `"list"` → `list(string)`
- Proxy provider blocks (provider blocks with only `alias`) in shared modules → use `configuration_aliases` in `required_providers`

### Environment Variables
- `HTTP_PROXY` no longer used for HTTPS connections → must set `HTTPS_PROXY` explicitly

### Platform Requirements
- Windows: UTF-8 and virtual terminal support required
- Windows 10 original release and Windows 8/earlier no longer supported

### Behavioral Changes
- Interrupting execution now returns non-zero exit status
- Provider-defined sensitive attributes are redacted in plan output

## Automated Fixes

### Replace list() and map() functions
```
Grep pattern: [^a-z_]list\(  in *.tf files
```
Replace `list(a, b, c)` with `tolist([a, b, c])`.

```
Grep pattern: [^a-z_]map\(  in *.tf files
```
Replace `map(k1, v1, k2, v2)` with `tomap({k1 = v1, k2 = v2})`.
**Careful**: do not replace `map(string)` type constraints — those are valid.

### Replace ignore_changes wildcard
```
Grep pattern: ignore_changes\s*=\s*\["\*"\]  in *.tf files
```
Replace with `ignore_changes = all`.

### Replace quoted type constraints
```
Grep pattern: type\s*=\s*"(map|list|string|number|bool)"  in *.tf files
```
Remove quotes: `type = "map"` → `type = map(string)`, `type = "list"` → `type = list(string)`.

### Replace -force flag in scripts
```
Grep pattern: terraform destroy.*-force  in *.sh, Makefile, *.yml, *.yaml, *.tf files
```
Replace `-force` with `-auto-approve`.

### Replace trailing DIR arguments in scripts
```
Grep pattern: terraform (plan|apply|destroy|validate|init)\s+[^\-]  in *.sh, Makefile, *.yml, *.yaml files
```
Review each match — if the trailing argument is a directory, replace with `-chdir=DIR`.

### Fix gcs backend path
```
Grep pattern: backend\s+"gcs"  in *.tf files
```
If found, check for `path` argument and replace with `prefix`.

### Fix HTTPS_PROXY in scripts/CI
```
Grep pattern: HTTP_PROXY  in *.sh, *.yml, *.yaml, Makefile, Dockerfile files
```
Ensure `HTTPS_PROXY` is set alongside `HTTP_PROXY` if Terraform makes HTTPS connections.

### Replace proxy provider blocks with configuration_aliases
```
Grep pattern: configuration_aliases  in *.tf files
```
If modules use provider blocks with only `alias` set, migrate to `configuration_aliases` in `required_providers`.

## Manual Steps

### Removed provisioners (chef, habitat, puppet, salt-masterless)
```
Grep pattern: provisioner\s+"(chef|habitat|puppet|salt-masterless)"  in *.tf files
```
If found, the developer must decide the replacement strategy:
- `local-exec` provisioner calling the tool's CLI
- Move configuration management to a separate pipeline (Ansible, etc.)
- Use a community provider

### Atlas backend migration
If using the `atlas` backend, the developer must choose a replacement backend (S3, GCS, Terraform Cloud, etc.).

## Verification

```bash
terraform fmt -check
terraform validate
terraform plan
```
