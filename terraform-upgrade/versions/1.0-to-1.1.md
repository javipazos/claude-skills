# Terraform 1.0 → 1.1

Introduces `moved` blocks and the `cloud` block for Terraform Cloud integration.

## Breaking Changes

### Removed CLI Options
- `terraform graph -type=validate` and `-type=eval` removed — use the plan graph instead

### Stricter Plugin Verification
- `terraform apply` with a saved plan now verifies provider plugin checksums match between plan and apply
- Custom workflows that modify plugin files between plan and apply will fail

### State Command Behavior
- `terraform state mv` errors when `-backup` or `-backup-out` used without `-state` on non-local backends (previously silently ignored)

### AzureRM Backend
- New opt-in `use_microsoft_graph` flag (will become default in 1.2) — start testing MSAL authentication now

### Platform Requirements
- macOS 10.13 High Sierra+

## Automated Fixes

### Remove -type=validate or -type=eval from graph commands in scripts
```
Grep pattern: terraform graph.*-type=(validate|eval)  in *.sh, Makefile, *.yml, *.yaml files
```
Remove the `-type` flag or switch to `-type=plan`.

## Manual Steps

### AzureRM backend users
If using the AzureRM backend, test with `use_microsoft_graph = true` now. It becomes the default in 1.2.

### Plugin workflow review
If you have custom automation that modifies provider binaries between `terraform plan` and `terraform apply`, it needs adjustment.

## Verification

```bash
terraform validate
terraform plan
```
