# Terraform 1.1 → 1.2

TLS hardening and AzureRM authentication default change.

## Breaking Changes

### TLS v1.2 Minimum
- Terraform CLI now requires TLS v1.2 minimum for all HTTPS connections
- TLS v1.0 and v1.1 are rejected
- Applies to Terraform CLI only, not provider plugins

### SHA-1 CA Certificates Rejected
- CA certificates signed with SHA-1 are no longer accepted
- Internal registries or custom backends using SHA-1 certs must reissue

### Built-in Credential Environment Variables
- `TF_TOKEN_` environment variables now built-in
- If using the `terraform-credentials-env` plugin, disable it before upgrading to avoid conflicts
- The built-in approach prioritizes env vars over credentials blocks and stored credentials (different from the plugin behavior)

### AzureRM Backend Defaults to MSAL
- The `use_microsoft_graph` option now defaults to `true`
- ADAL (Azure AD Graph) authentication is still available but deprecated

### macOS TLS Certificate Validation
- macOS now uses platform APIs for TLS cert validation — may change which root certificates are accepted

### Platform Requirements
- Linux kernel 2.6.32+

## Automated Fixes

### Remove terraform-credentials-env plugin references
```
Grep pattern: terraform-credentials-env  in *.tf, *.hcl, .terraformrc, terraform.rc files
```
Remove plugin configuration if present. The built-in `TF_TOKEN_` support replaces it.

### Update AzureRM backend if explicitly setting use_microsoft_graph
```
Grep pattern: use_microsoft_graph  in *.tf files
```
If explicitly set to `false`, this should be migrated to `true` or removed (defaults to `true` now).

## Manual Steps

### TLS certificate compatibility
If connecting to internal services with TLS 1.0/1.1 only or SHA-1 certificates, those services must be upgraded. The developer or infrastructure team must handle this.

### hashicorp/tfe provider version
If using the `hashicorp/tfe` provider for Terraform Cloud/Enterprise management, upgrade to v0.31+ for compatibility with the built-in `TF_TOKEN_` variables.
