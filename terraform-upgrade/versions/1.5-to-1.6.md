# Terraform 1.5 → 1.6

Major S3 backend restructuring. OS minimum versions raised.

## Breaking Changes

### Platform Requirements
- macOS 10.15 Catalina+ required
- Windows 10 / Windows Server 2016+ required

### S3 Backend Restructured

IAM role configuration moved to nested `assume_role` block:
```hcl
# Before (deprecated)
backend "s3" {
  role_arn                  = "arn:aws:iam::123456789012:role/my-role"
  session_name              = "my-session"
  external_id               = "my-external-id"
  assume_role_duration_seconds = 3600
  assume_role_policy_arns   = ["arn:aws:iam::123456789012:policy/my-policy"]
  assume_role_tags          = { Environment = "prod" }
  assume_role_transitive_tag_keys = ["Environment"]
}

# After
backend "s3" {
  assume_role {
    role_arn                = "arn:aws:iam::123456789012:role/my-role"
    session_name            = "my-session"
    external_id             = "my-external-id"
    duration                = "1h"
    policy_arns             = ["arn:aws:iam::123456789012:policy/my-policy"]
    tags                    = { Environment = "prod" }
    transitive_tag_keys     = ["Environment"]
  }
}
```

Endpoint configuration moved to nested `endpoints` block:
```hcl
# Before (deprecated)
backend "s3" {
  dynamodb_endpoint = "http://localhost:4566"
  iam_endpoint      = "http://localhost:4566"
  endpoint          = "http://localhost:4566"
  sts_endpoint      = "http://localhost:4566"
}

# After
backend "s3" {
  endpoints {
    dynamodb = "http://localhost:4566"
    iam      = "http://localhost:4566"
    s3       = "http://localhost:4566"
    sts      = "http://localhost:4566"
  }
}
```

Other deprecated arguments:
- `shared_credentials_file` → `shared_credentials_files` (list)
- `force_path_style` → `use_path_style`

Deprecated environment variables:
- `AWS_DYNAMODB_ENDPOINT` → `AWS_ENDPOINT_URL_DYNAMODB`
- `AWS_IAM_ENDPOINT` → `AWS_ENDPOINT_URL_IAM`
- `AWS_S3_ENDPOINT` → `AWS_ENDPOINT_URL_S3`
- `AWS_STS_ENDPOINT` → `AWS_ENDPOINT_URL_STS`

## Automated Fixes

### Restructure S3 backend assume_role
```
Grep pattern: backend\s+"s3"  in *.tf files
```
If found, check for top-level `role_arn`, `session_name`, `external_id`, `assume_role_duration_seconds`, `assume_role_policy_arns`, `assume_role_tags`, `assume_role_transitive_tag_keys`. Move them into an `assume_role {}` block with the renamed arguments.

### Restructure S3 backend endpoints
Check for top-level `dynamodb_endpoint`, `iam_endpoint`, `endpoint`, `sts_endpoint`. Move them into an `endpoints {}` block with renamed keys.

### Rename S3 backend arguments
- `shared_credentials_file = "path"` → `shared_credentials_files = ["path"]`
- `force_path_style = true` → `use_path_style = true`

### Update environment variables in scripts/CI
```
Grep pattern: AWS_DYNAMODB_ENDPOINT|AWS_IAM_ENDPOINT|AWS_S3_ENDPOINT|AWS_STS_ENDPOINT  in *.sh, *.yml, *.yaml, Makefile, Dockerfile, *.env files
```
Replace with the new `AWS_ENDPOINT_URL_*` names.

## Manual Steps

### S3 backend migration
After restructuring the backend config, run `terraform init -reconfigure` to apply the new configuration. This does not migrate state, it just reconfigures the backend client.

## Verification

```bash
terraform init -reconfigure   # if S3 backend was changed
terraform validate
terraform plan
```
