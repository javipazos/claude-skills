# Terraform 1.2 → 1.3

Several backends removed, ADAL authentication removed, optional object attributes introduced.

## Breaking Changes

### Removed Backends
The following backends are removed entirely:
- `artifactory`
- `etcd`
- `etcdv2`
- `etcdv3`
- `manta`
- `swift`
- Legacy `azure` backend name (use `azurerm` instead)

### AzureRM Backend — ADAL Removed
- MSAL (Microsoft Graph) is now the only authentication method
- `use_microsoft_graph` argument removed — MSAL is always used
- If not already migrated from ADAL, this is a hard blocker

### TLS Handshake Validation
- Rejects TLS handshakes with duplicate extensions per RFC 5246 / RFC 8446
- May cause errors with misconfigured TLS servers

### terraform import Changes
- `-allow-missing-config` flag removed
- Imported resources now require at least an empty configuration block

### JSON Output Format
- `after_unknown` in `terraform show -json` now provides detailed structure for complex values instead of simple true/false

## Automated Fixes

### Check for removed backends
```
Grep pattern: backend\s+"(artifactory|etcd|etcdv2|etcdv3|manta|swift)"  in *.tf files
```
If found, flag as a **blocking manual step** — the developer must choose a replacement backend.

### Replace legacy azure backend name
```
Grep pattern: backend\s+"azure"  in *.tf files
```
Replace with `backend "azurerm"`.

### Remove -allow-missing-config from import scripts
```
Grep pattern: terraform import.*-allow-missing-config  in *.sh, Makefile, *.yml, *.yaml files
```
Remove the flag and ensure each imported resource has a configuration block.

### Remove use_microsoft_graph from AzureRM backend
```
Grep pattern: use_microsoft_graph  in *.tf files
```
Remove the argument — MSAL is now the only option.

## Manual Steps

### Backend migration (if using removed backends)
If the codebase uses any of the removed backends, the developer must:
1. Choose a replacement (S3, GCS, AzureRM, Terraform Cloud, etc.)
2. Migrate state with `terraform init -migrate-state`
3. This requires careful planning and cannot be automated safely

### ADAL authentication migration
If the AzureRM backend was still using ADAL, ensure service principal credentials and Azure AD app registrations are configured for Microsoft Graph API permissions.
