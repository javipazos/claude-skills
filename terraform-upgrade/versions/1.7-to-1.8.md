# Terraform 1.7 → 1.8

Provider-defined functions and plan output changes.

## Breaking Changes

### Plan Presentation for Lists
- `terraform plan` now compares same-length list changes element-by-element (by index) instead of showing the whole list diff
- Different-length lists retain previous behavior
- This is a display change only — no functional difference

### JSON Plan Output
- `terraform show -json` now includes `applyable` and `complete` flags
- Tools parsing JSON plan output should handle these new fields (but they are additive)

### Sensitive Value Tracking
- More precise sensitive value tracking in state and plans
- May cause values that were previously shown in plan output to now appear as `(sensitive value)`

## Automated Fixes

No configuration changes typically needed. These changes are behavioral/output formatting.

### Check for JSON plan output parsing in CI/CD
```
Grep pattern: terraform show -json|terraform show.*-json  in *.sh, Makefile, *.yml, *.yaml files
```
If found, flag for developer review — new fields are present but shouldn't break existing parsers.

## Manual Steps

### Provider-defined functions
Providers can now expose custom functions via `provider::provider_name::function_name()`. No action needed unless you want to adopt them. The built-in `terraform` provider adds `decode_tfvars`, `encode_tfvars`, and `encode_expr`.

### moved blocks across resource types
`moved` blocks now support moving resources between different resource types. Useful for migrating `null_resource` to `terraform_data`.

## Verification

```bash
terraform validate
terraform plan
```
