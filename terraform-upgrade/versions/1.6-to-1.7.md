# Terraform 1.6 → 1.7

State interoperability fix and S3 backend credential behavior change.

## Breaking Changes

### State Interoperability Issue
Users who need to share state across different Terraform minor versions must be on specific patch releases:
- Terraform 1.3.x → must use **1.3.10+**
- Terraform 1.4.x → must use **1.4.7+**
- Terraform 1.5.x → must use **1.5.7+**
- Terraform 1.6.x and 1.0-1.2.x are unaffected

This matters for teams using `terraform_remote_state` data sources across workspaces running different versions. If other team members or CI pipelines run older patch versions, coordinate the upgrade.

### S3 Backend Credential Ordering
- `use_legacy_workflow` now defaults to `false`
- Credential search order now matches AWS SDK/CLI defaults
- To keep the old behavior, explicitly set `use_legacy_workflow = true` (but this is deprecated and will be removed)

### terraform graph Simplified
- Now produces a simplified resources-only graph by default
- Previous detailed dependency graph available via `-type=plan`

### nonsensitive() Behavior
- `nonsensitive()` no longer errors when applied to values already marked non-sensitive
- This is a relaxation, not a breaking change for most users

## Automated Fixes

### Check for use_legacy_workflow in S3 backend
```
Grep pattern: backend\s+"s3"  in *.tf files
```
If using S3 backend, check if credential ordering matters. If the backend works without `use_legacy_workflow`, no change needed.

### Update terraform graph usage in scripts
```
Grep pattern: terraform graph  in *.sh, Makefile, *.yml, *.yaml files
```
If scripts depend on the detailed graph output, add `-type=plan`.

## Manual Steps

### Coordinate version across team
If your team uses multiple Terraform versions across different workspaces or CI pipelines, verify all members are on compatible patch versions (see the interoperability table above).

### S3 credential ordering
If your S3 backend authentication relies on a specific credential resolution order that differs from AWS SDK defaults, test thoroughly. You may need to adjust IAM configuration or explicitly set credential sources.

## Verification

```bash
terraform validate
terraform plan
```
