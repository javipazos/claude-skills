# Terraform 1.9 → 1.10

S3 backend deprecated attributes removed, ephemeral resources introduced.

## Breaking Changes

### S3 Backend — Deprecated assume_role Attributes Removed
Top-level assume_role attributes that were deprecated in 1.6 are now **removed**:
- `role_arn`, `session_name`, `external_id`, `assume_role_duration_seconds`
- `assume_role_policy_arns`, `assume_role_tags`, `assume_role_transitive_tag_keys`

These **must** use the `assume_role {}` nested block. If the 1.5→1.6 migration was done correctly, no action needed. If deprecation warnings were ignored, this is now a hard error.

### Moved Block Reserved Keywords
- `moved` blocks now respect reserved keywords when parsing resource addresses
- Resources with type names matching top-level blocks (e.g., `resource`, `variable`, `output`, `data`, `module`, `locals`, `terraform`, `provider`) must be prefixed with `resource.` in moved block addresses

### S3 Native State Locking
- S3 backend now supports native state locking (alongside DynamoDB)
- When both are configured, locks are acquired from both sources
- DynamoDB-based locking will be deprecated in a future version

### Sensitive Marks in Conditionals
- Conditional expressions must now combine sensitivity marks from all branches
- Previously non-sensitive results from conditionals may now appear as sensitive if any branch contains sensitive values

### Deprecated CLI Flags
- `-state` flag for `plan`, `apply`, and `refresh` commands produces deprecation warnings
- Use the `path` attribute in the `local` backend configuration instead

## Automated Fixes

### Check for removed S3 top-level assume_role attributes
```
Grep pattern: backend\s+"s3"  in *.tf files
```
If found, verify no top-level `role_arn`, `session_name`, etc. remain. If they do, move them into the `assume_role {}` block (same migration as 1.5→1.6 but now mandatory).

### Check for moved blocks with reserved keywords
```
Grep pattern: moved\s*\{  in *.tf files
```
Check if any `from` or `to` addresses use resource type names that match reserved words. Prefix with `resource.` if needed.

### Check for -state flag in scripts
```
Grep pattern: terraform (plan|apply|refresh).*-state  in *.sh, Makefile, *.yml, *.yaml files
```
Flag for developer review — should migrate to local backend path configuration.

## Manual Steps

### S3 native state locking
If currently using DynamoDB for state locking, the developer may want to plan a migration to S3 native locking in a future version. No immediate action required.

### Ephemeral resources
New experimental feature — ephemeral resources are read fresh each evaluation and never persisted to state. No action needed unless adopting.

## Verification

```bash
terraform init -upgrade   # if S3 backend was changed
terraform validate
terraform plan
```
