# Terraform 1.12 → 1.13

Test variable definitions now required, empty workspace names rejected.

## Breaking Changes

### Test External Variables
- External variables referenced in `.tftest.hcl` files must now have a `variable` definition block within the test file
- Users with complex external variables may see errors without the additional definition

### Workspace Name Validation
- Empty string workspace names (`""`) are now rejected by validation and `workspace delete`

### Module Output Sensitivity
- Nested module outputs now properly retain sensitivity markings as specified in configuration
- May cause outputs that were previously visible in plan to now appear as `(sensitive value)`

## Automated Fixes

### Check test files for external variable references
```
Grep pattern: var\.  in *.tftest.hcl files
```
For each variable reference, check if a corresponding `variable` block exists in the test file. If not, add one:
```hcl
variable "name" {
  type = string  # match the type from the main configuration
}
```

### Check for empty workspace names in scripts
```
Grep pattern: terraform workspace.*""  in *.sh, Makefile, *.yml, *.yaml files
```
If found, fix to use a valid workspace name.

## Manual Steps

### Test variable types
When adding `variable` blocks to test files, the developer should verify the types match the main configuration. The agent can propose types based on usage, but the developer should confirm.

## Verification

```bash
terraform validate
terraform test  # if test files exist
terraform plan
```
