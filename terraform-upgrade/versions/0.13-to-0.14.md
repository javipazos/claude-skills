# Terraform 0.13 → 0.14

Primary changes: dependency lock file introduced, sensitive output enforcement, TLS hardening.

## Recommended Tool

```bash
terraform init
terraform fmt
```

No dedicated upgrade command for 0.14. The `terraform 0.13upgrade` command no longer exists in this version.

Run `terraform init` to generate the new `.terraform.lock.hcl` file.

## Breaking Changes

### Dependency Lock File
- `terraform init` now generates `.terraform.lock.hcl` tracking provider checksums
- This file should be committed to version control
- To always use newest allowed versions (previous behavior), run `terraform init -upgrade`

### Sensitive Output Enforcement
- Outputs that reference sensitive values must be marked `sensitive = true`
- Terraform errors at plan time if a non-sensitive output references a sensitive variable, sensitive module output, or provider-defined sensitive attribute
- This is a hard error, not a warning

### Provider version Argument Deprecated
- The `version` argument inside `provider` blocks now produces deprecation warnings
- Move version constraints to `required_providers` in the `terraform` block

### TLS Hardening
- Certificate hostname verification no longer accepts Common Name — must use Subject Alternative Names (SAN)
- RFC 8446 client-side TLS downgrade protection enforced
- HTTP header parsing stricter: ASCII whitespace only, `Transfer-Encoding: identity` disallowed

### Removed Features
- `terraform 0.13upgrade` command removed
- `debug` command removed

### Platform Requirements
- macOS 10.12 Sierra+

## Automated Fixes

### Mark sensitive outputs
```
Grep pattern: sensitive\s*=\s*true  in *.tf variable blocks
```
Find all sensitive variables, then trace which outputs reference them:
```
Grep pattern: output\s+"  in *.tf files
```
For outputs referencing sensitive values, add `sensitive = true` to the output block.

### Move provider version constraints
```
Grep pattern: version\s*=  inside provider blocks in *.tf files
```
Move the version constraint from the `provider` block to `required_providers`:
```hcl
# Before (deprecated)
provider "aws" {
  version = "~> 3.0"
  region  = "us-east-1"
}

# After
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 3.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}
```

## Manual Steps

### TLS certificate compatibility
If Terraform connects to internal services (private registries, custom backends) using certificates with only Common Name (no SAN), those certificates must be reissued. The developer or infrastructure team must handle this.

### Review lock file strategy
The developer should decide whether to commit `.terraform.lock.hcl` to version control (recommended) or add it to `.gitignore`.

## Verification

```bash
terraform fmt -check
terraform validate
terraform plan
```

Check that `.terraform.lock.hcl` was generated.
