# Terraform 1.4 → 1.5

Introduces `import` blocks and `check` blocks. Last version supporting older OS versions.

## Breaking Changes

### Platform Support Ending After 1.5
- This is the **last version** supporting macOS 10.13 High Sierra and 10.14 Mojave
- This is the **last version** supporting Windows 7, 8, Server 2008, and Server 2012
- If running on these platforms, you cannot upgrade beyond 1.5 without upgrading the OS

### DNS Resolution (Linux)
- Linux now respects the `trust-ad` flag in `/etc/resolv.conf`
- Sets the "authentic data" flag in DNS requests when enabled
- May produce different DNS responses than previous versions

## Automated Fixes

No configuration changes required for this upgrade. The new features (`import` blocks, `check` blocks, `-generate-config-out`) are additive.

### Check platform compatibility for next upgrade
If the developer's target version is beyond 1.5, verify:
```bash
uname -s  # OS
sw_vers 2>/dev/null || cat /etc/os-release 2>/dev/null  # OS version
```
Warn if the platform won't be supported in 1.6+.

## Manual Steps

### OS upgrade planning
If running on macOS <10.15 or Windows <10, the developer must plan an OS upgrade before proceeding beyond 1.5.

## Verification

```bash
terraform validate
terraform plan
```
