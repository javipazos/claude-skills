# Terraform 1.13 → 1.14

Minor release within the stable v1.0 series.

## Breaking Changes

### Container Runtime Parallelism
- Parallelism of Terraform operations may be reduced in container runtimes depending on CPU bandwidth limit settings
- Affects Docker, Kubernetes pods, and other containerized environments with CPU limits

### Build Requirements (source builds only)
- Building Terraform from source now requires macOS Monterey (12.0) or later
- Only affects developers building Terraform from source, not users of pre-built binaries

## Automated Fixes

No configuration changes needed.

### Check CI/CD for container resource limits
```
Grep pattern: (cpu|CPU|resources|limits)  in Dockerfile, *.yml, *.yaml, docker-compose*.yml files
```
If Terraform runs in containers with CPU limits, parallelism may be reduced. The developer may want to adjust `terraform plan -parallelism=N` or container resource limits.

## Manual Steps

### Container performance review
If running Terraform in CI/CD containers with tight CPU limits, benchmark plan/apply times after upgrading. Adjust `-parallelism` flag or container resources if performance degrades.

## Verification

```bash
terraform validate
terraform plan
```
